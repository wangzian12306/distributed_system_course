# 商品库存与秒杀系统设计

## 1. 系统架构草图

### 1.1 服务拆分

```
+------------------+     +------------------+     +------------------+
|                  |     |                  |     |                  |
|   用户服务       |<--->|   订单服务       |<--->|   商品服务       |
| User Service     |     | Order Service    |     | Product Service  |
|                  |     |                  |     |                  |
+------------------+     +------------------+     +------------------+
          |                        |                        |
          v                        v                        v
+------------------+     +------------------+     +------------------+
|                  |     |                  |     |                  |
|   认证中心       |     |   库存服务       |     |   秒杀服务       |
| Auth Center      |     | Inventory Service|     | Flash Sale Service|
|                  |     |                  |     |                  |
+------------------+     +------------------+     +------------------+
          |                        |                        |
          v                        v                        v
+------------------+     +------------------+     +------------------+
|                  |     |                  |     |                  |
|   MySQL数据库    |     |   Redis缓存      |     |   MQ消息队列     |
| MySQL Database   |     | Redis Cache      |     | Message Queue    |
|                  |     |                  |     |                  |
+------------------+     +------------------+     +------------------+
```

### 1.2 核心流程

1. **用户注册/登录**：用户通过用户服务进行注册和登录，获取认证令牌
2. **商品浏览**：用户通过商品服务浏览商品信息，包括秒杀商品
3. **秒杀活动**：用户在秒杀服务中参与秒杀活动
4. **订单创建**：秒杀成功后，订单服务创建订单
5. **库存扣减**：订单创建时，库存服务进行库存扣减（使用Redis分布式锁确保原子性）
6. **消息通知**：通过消息队列异步处理订单通知等操作

## 4. 技术栈选型说明

### 4.1 核心技术栈

| 技术 | 版本 | 用途 | 选型理由 |
|------|------|------|----------|
| Spring Boot | 2.7.x | 应用框架 | 快速构建微服务应用，简化配置，提供丰富的starter |
| MyBatis | 3.5.x | ORM框架 | 轻量级ORM框架，灵活的SQL映射，支持动态SQL |
| MySQL | 8.0+ | 关系型数据库 | 稳定可靠，适合存储结构化数据，支持事务和锁机制 |
| Redis | 7.0+ | 缓存和分布式锁 | 用于缓存热点数据，实现分布式锁确保秒杀库存扣减的原子性 |
| RabbitMQ | 3.8+ | 消息队列 | 用于异步处理订单通知等操作，削峰填谷 |
| Spring Cloud | 2021.x | 微服务框架 | 提供服务注册与发现、负载均衡、熔断等功能 |
| Spring Security | 5.7.x | 安全框架 | 实现用户认证和授权 |

### 4.2 秒杀系统特殊考虑

1. **高并发处理**：
   - 使用Redis缓存秒杀商品信息和库存
   - 利用Redis的原子操作进行库存扣减
   - 采用分布式锁防止超卖

2. **防止重复下单**：
   - 使用Redis的Set结构记录已参与秒杀的用户
   - 对同一用户的重复请求进行限流

3. **系统降级**：
   - 当系统负载过高时，返回友好的错误信息
   - 优先保证核心功能的可用性

4. **数据一致性**：
   - 使用消息队列异步处理订单创建
   - 定期对账确保库存和订单数据一致性

### 4.3 部署方案

- **容器化**：使用Docker容器化部署各服务
- **编排**：使用Kubernetes进行服务编排和管理
- **监控**：集成Prometheus和Grafana进行系统监控
- **日志**：使用ELK stack进行日志收集和分析

## 5. 系统优化建议

1. **数据库优化**：
   - 对高频查询的字段建立索引
   - 使用数据库连接池
   - 分库分表处理大量订单数据

2. **缓存优化**：
   - 合理设置缓存过期时间
   - 使用缓存预热减少冷启动时间
   - 实现缓存击穿、缓存穿透和缓存雪崩的防护措施

3. **代码优化**：
   - 使用异步处理非核心业务逻辑
   - 优化SQL语句，减少数据库压力
   - 使用线程池处理并发请求

4. **架构优化**：
   - 服务拆分合理，职责单一
   - 实现服务降级和熔断机制
   - 考虑使用读写分离提高数据库性能
